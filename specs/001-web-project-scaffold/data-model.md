# Data Model: Web Project Scaffold

**Feature**: 001-web-project-scaffold  
**Phase**: 1 — Design  
**Date**: 2026-05-02

---

> This feature has no database or persistent data. The entities below describe the **file and folder structure** of the project — the things Joey will create, own, and maintain.

---

## Entities

### Project Root

The top-level folder of the repository. Contains all configuration files. Joey should never need to edit these files to add new pages or components — they are set up once and left alone.

**Files at root**:

| File | Purpose |
|---|---|
| `package.json` | Lists dependencies and defines run commands (`dev`, `build`, `preview`) |
| `tsconfig.json` | TypeScript project references config |
| `tsconfig.app.json` | TypeScript settings for source code — strict mode, path aliases |
| `vite.config.ts` | Vite build settings — React plugin, `@/*` path alias |
| `index.html` | HTML entry point — Vite injects the built JS here |
| `tailwind.config.js` | Tailwind CSS settings — includes `src/` in content scan |
| `postcss.config.mjs` | PostCSS settings — required by Tailwind, not edited directly |
| `components.json` | shadcn/ui settings — style, paths, aliases |
| `.gitignore` | Files and folders Git should ignore |
| `README.md` | Beginner-friendly setup and deployment instructions |
| `bun.lockb` | Bun's lock file — records exact dependency versions |

**Validation rules**: None. These are static config files created by tooling.

---

### Source Folder (`src/`)

Where Joey writes all application code. Everything inside `src/` is code Joey will actively read, edit, and add to.

**Subfolders**:

| Folder/File | Purpose |
|---|---|
| `src/main.tsx` | App entry point — mounts `<App />` into `index.html`. Joey doesn’t edit this often. |
| `src/App.tsx` | Root component — the starter page. This is where Joey starts coding. |
| `src/index.css` | Global styles — Tailwind directives + shadcn/ui CSS variables |
| `src/components/ui/` | shadcn/ui component files — auto-generated by the shadcn CLI |
| `src/lib/` | Shared utilities — starts with `utils.ts` from shadcn setup |

---

### Starter Page

The default landing page shown when the dev server or live site loads. It lives at `src/App.tsx`.

**Fields**:

| Field | Type | Description |
|---|---|---|
| Heading | `string` | Display title of the page |
| Description | `string` | One-sentence explanation of what the project is |
| Button | shadcn/ui `Button` | Demonstrates that shadcn/ui is installed and styled |

**Validation rules**:
- Must render without TypeScript errors
- Must show at least one shadcn/ui component (FR-003)
- Must apply Tailwind layout classes (centered, readable on mobile)

---

### shadcn/ui Components Folder

`src/components/ui/` — contains component files generated by running `bunx shadcn add <component>`. Joey does not write these files by hand; the CLI creates them. Joey can then import and use them anywhere in `src/`.

**State**: Starts with `Button` after initial setup. Grows as Joey adds more components.

---

### Global Styles

`src/index.css` — the single CSS file for the project. Contains Tailwind’s base directives and the CSS variable definitions for the shadcn/ui theme. Joey does not edit this file to change component styles; it is set up automatically.

## Relationships

```
Project Root
└── src/
    ├── app/
    │   ├── layout.tsx        ← wraps every page (imports globals.css)
    │   ├── page.tsx          ← Starter Page (imports Button from components/ui)
    │   └── globals.css       ← Global Styles
    ├── components/
    │   └── ui/
    │       └── button.tsx    ← shadcn/ui Components Folder (starts here)
    └── lib/
        └── utils.ts          ← shared cn() helper used by shadcn components
```

## State Transitions (Project Lifecycle)

| State | Description | How to get there |
|---|---|---|
| Fresh clone | No dependencies installed | `git clone` |
| Ready to run | Dependencies installed | `bun install` |
| Running locally | Dev server active | `bun run dev` |
| Deployed | Live on Vercel | Connect repo to Vercel, push to main |
| Extended | New shadcn/ui component added | `npx shadcn add <component>` |
