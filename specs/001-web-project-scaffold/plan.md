# Implementation Plan: Web Project Scaffold

**Branch**: `main` | **Date**: 2026-05-02 | **Spec**: [spec.md](spec.md)

## Summary

Initialize the project as a Vite + React SPA using Bun, with TypeScript, Tailwind CSS, and shadcn/ui configured. The result is a single starter page with one shadcn/ui Button component, deployable to Vercel with zero manual build configuration.

**Structure Decision**: Vite SPA with flat `src/` directory. Config files stay at root; Joey writes code only inside `src/`. The entry point (`main.tsx`) and root component (`App.tsx`) are immediately obvious — no special folder conventions to memorize. This satisfies FR-010 and the constitution’s “radically simple” principle.

## Implementation Steps

> These steps are expanded into tasks by `/speckit.tasks`. Listed here for plan completeness.

1. Scaffold Vite project with `bun create vite . --template react-ts`
2. Install Tailwind CSS and configure it for Vite (`bun add -d tailwindcss postcss autoprefixer` + init)
3. Add `@/*` path alias to `vite.config.ts` and `tsconfig.app.json` (required by shadcn/ui)
4. Run `bunx shadcn@latest init` (New York style, Neutral color, CSS variables)
5. Run `bunx shadcn@latest add button`
6. Replace `src/App.tsx` with the starter page content (heading, description, Button)
7. Verify `bun run build` exits with code 0 and zero TypeScript errors
8. Update `README.md` with beginner-friendly setup instructions
9. Push to GitHub and connect to Vercel — confirm deployment succeeds
